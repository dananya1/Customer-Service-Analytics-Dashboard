**Overview**

This project focuses on analyzing customer service data using MS Excel to evaluate service demand, customer satisfaction, call duration, purchase behavior, and representative performance. The goal was to build an interactive dashboard that tracks service KPIs and helps understand customer support workload patterns.

**Business Problem**

Customer support teams need visibility into call demand, service performance, and operational workload in order to plan staffing and improve service delivery. This project aimed to identify peak call periods, high-volume locations, and representative-level performance patterns to support better operational planning.

**Objectives**

1.Analyze customer service interactions and representative performance

2.Identify peak call periods and high-volume service locations

3.Track customer service KPIs in an interactive dashboard

4.Improve reporting efficiency for support operations

**Tools Used**

MS Excel, Pivot Tables, Power Query, VLOOKUP, Charts

**Dataset Details**

1.Records analyzed: 1,000+ customer service records

2.Key fields used:

Call date / day, 
City / location, 
Call duration, 
Customer satisfaction, 
Purchase behavior, 
Representative performance

**Steps Performed**

1.Cleaned and organized raw service data using Power Query

2.Used Pivot Tables and VLOOKUP to summarize service metrics and prepare KPI views

Built an interactive Excel dashboard to monitor:

customer satisfaction, 
call duration, 
representative performance,
purchase behavior, 
daily and city-level call trends

**Key Insights**

1.Saturday was identified as the peak call day with 161 calls

2.Cleveland recorded the highest call volume with 389 calls

3.Service data analysis helped highlight demand patterns and workload concentration across support operations
